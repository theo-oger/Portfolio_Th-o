Le challenge consiste à analyser un fichier image disque nommé usb.image plutôt que de chercher à le monter ou à l’exécuter on peut simplement l’ouvrir avec un éditeur de texte comme VS Code ou même un hexadump pour voir son contenu brut.

En parcourant les lignes on remarque rapidement qu il contient des métadonnées au format RDF et à la ligne 14 on trouve directement l information recherchée le nom du propriétaire sous la balise.

< rdf:li > Prénom NOM < /rdf:li >

La solution ici n est pas forcément la plus élégante mais elle reste efficace on utilise simplement la commande strings pour extraire toutes les chaînes lisibles du fichier mémoire ch2.dmp

strings ch2.dmp > resultat.txt

On obtient alors un fichier texte contenant toutes les chaînes ASCII présentes dans le dump il suffit ensuite de parcourir ce fichier et de chercher les occurrences liées à Netbios puisque c est là que l on peut trouver le nom de la machine.

En fouillant un peu on tombe sur la ligne suivante :

\Device\NetbiosSmbWIN-ETSA91RKCFPWORKGROUP

Le principe de ce challenge repose sur la compréhension du rôle de GRUB qui est le programme d’amorçage permettant de choisir sur quel système on va démarrer. GRUB est stocké dans la partition /boot qui est séparée du système d’exploitation et qui, en général, n’est pas chiffrée par les outils de chiffrement de disque. Cela ouvre la possibilité de modifier directement ses paramètres pour influencer le démarrage.

Lors du boot, quand on arrive sur l’écran GRUB, il suffit d’appuyer sur la touche e pour éditer les options de lancement. On repère la ligne qui contient les paramètres du noyau et c’est là qu’on intervient.

On remplace le paramètre ro (read-only) par rw (read-write) afin d’obtenir les permissions d’écriture. Ensuite, on ajoute l’option init=/bin/bash qui indique au noyau de démarrer directement sur un shell bash.

Avec ces modifications, au prochain boot, la machine démarre sur un shell root avec les droits d’écriture, ce qui permet d’accéder au système sans passer par l’authentification classique.
