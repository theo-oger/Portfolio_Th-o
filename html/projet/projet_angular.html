Projet GSB — NodeJS

Ce projet a été réalisé lors de ma deuxième année de BTS SIO, en Atelier Professionnel (AP). Un ancien élève de Turgot est venu nous présenter le fonctionnement de NodeJS et ses différentes possibilités. À la suite de cette présentation, nous avons entrepris la réalisation d'un projet concret. L'objectif était de concevoir une application de type GSB (Gestion des Soins et des Bénéficiaires), mettant en scène des docteurs, des clients et des médicaments.

Le projet consistait à utiliser les bases de NodeJS pour structurer une application web dynamique, en manipulant les routes, les modèles et les interactions entre les différents acteurs du système. Nous avons développé une API REST permettant d'interroger une base de données, ainsi que d'effectuer des opérations CRUD (Create, Read, Update, Delete) sur les différentes entités, notamment l'ajout et la suppression de docteurs.

Fonctionnalités développées

L'application développée comprend plusieurs fonctionnalités clés :

Nous avons mis en place une API RESTful avec NodeJS permettant de :

- Interroger la base de données pour récupérer les informations des docteurs, clients et médicaments
- Ajouter de nouveaux docteurs dans le système avec leurs informations complètes
- Supprimer des docteurs existants de la base de données
- Gérer les relations entre les différentes entités du système

Cette API constitue le cœur de l'application et permet une interaction dynamique avec les données.

État du projet

Le projet a permis de découvrir les fondamentaux de NodeJS et de mettre en pratique la création d'une application web structurée avec une architecture API REST. De nouvelles fonctionnalités pourraient être ajoutées à l'avenir, notamment la gestion avancée des prescriptions, la sécurisation des accès via authentification et une interface utilisateur plus ergonomique.
